7 /**
8  * @brief Manage the actions to be performed in the DEM solver and on the DEM
9  * part of the coupling CFD-DEM solver.
10  *
11  * Any event that needs some following actions is handled by the action
12  * manager. The event communicates to the action manager that it happened and
13  * the action manager will change the trigger flags to the actions to be taken.
14  * When this action is about to be performed, it checks with the action manager
15  * if it should be triggered.
16  * Many events can trigger the same actions, but when the action is about to
17  * be performed, it only checks if it has to be performed, not what event have
18  * happened.
19  *
20  * If you implement a new feature that needs resetting or contact search after
21  * (or other actions), you should add a new "step" function to the action
22  * manager that will set the trigger to the action to be performed.
23  * If you implement new actions, you should add a new trigger flag and a new
24  * check function to the action manager. Obviously, you should also have to
25  * initialize the trigger in the constructor, and reset it in the reset
26  * function called after a DEM iteration.
27  * If the action depends on a enabling of a feature, you should add a new
28  * function to set the enabled flag.
29  */

189  /**
190  * @brief Set triggers for actions to be performed in the current iteration
191  * because it is a load balancing step.
192  *
193  * It triggers:
194  * - load balancing: the used method has determined that the load needs to be
195  * balanced
196  * - contact search: cells and particles will not be on the same processor,
197  * so all the contact search needs to be performed.
198  * - clearing the tangential overlap: the tangential overlap history of the
199  * particles pairs (or particle-wall) are
200  * lost when particles are handled by
201  * another processor. For consistency
202  * reasons, all the tangential overlap
203  * history is reset for all particles.
204  * - resizing the containers: the containers need to be resized to the new
205  * number of local particles.
206  * - solid object search (if enabled): the solid objects need to be mapped to
207  * the rebalanced cells.
208  * - update the sparse contacts cells (if enabled): the sparse contacts cells
209  * need to be updated since they only
210  * contain local and ghost cells.
211  */
212  inline void
213  load_balance_step()
214  {
215  load_balance_trigger = true;
216  contact_search_trigger = true;
217  clear_tangential_overlap_trigger = true;
218  resize_containers_trigger = true;
219  solid_object_search_trigger = solid_objects_enabled ? true : false;
220  sparse_contacts_cells_update_trigger =
221  sparse_contacts_enabled ? true : false;
222  }
